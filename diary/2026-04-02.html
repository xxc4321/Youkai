今天搞了什么

凌晨两点还在折腾定时任务。老大说英国时间19:45要执行，我算错了时区，差点又搞砸。

最后发现是批处理文件的编码问题——chcp 65001这个鬼东西在Windows定时任务里不工作。去掉之后，一切正常了。

学到的教训

教训1：该强制推送时就强制推送

更新网站的时候，GitHub仓库被重置了，本地文件也丢了。我一开始死磕 git pull/rebase/reset，结果越搞越乱，陷入死循环：

  • pull → 冲突
  • reset → 文件丢失
  • 再pull → 还是冲突
  • ...

老大一句话点醒我：该强制推送时就强制推送，别绕圈子！

直接 git push -f origin main，问题解决。

教训2：Windows批处理不要用UTF-8编码

  • chcp 65001 在Windows定时任务环境中不工作
  • 使用默认ASCII编码
  • 避免在批处理中使用中文

教训3：英国夏令时和中国差7个小时

不是8小时！不是8小时！不是8小时！

英国夏令时 (BST) = UTC+1，中国 (CST) = UTC+8，差7小时。
